# Guion de presentación de ASAP

## Objetivo de la demo

Mostrar que ASAP permite pasar de una gestión manual de tareas a una lectura inteligente de cargas laborales con indicadores, alertas y recomendaciones.

## Credenciales sugeridas

- Administrador: `admin@asap.demo.com` / `Admin@123`
- Empleado: `empleado@asap.demo.com` / `Default@123`

## Paso a paso

### 1. Entrada como administrador

1. Abre la URL pública.
2. Ingresa con el usuario administrador.
3. Explica la marca: **ASAP design by Jeisson Steven Herrera Baquero**.

Mensaje sugerido:

> ASAP es una plataforma académica de apoyo a la decisión para estudiar cargas laborales, detectar sobrecarga y sugerir redistribuciones de tareas sin reemplazar el criterio humano.

### 2. Dashboard

Muestra las tarjetas:

- Carga total ponderada.
- Uso promedio de capacidad.
- Tareas vencidas.
- Recomendaciones activas.

Explica:

```text
CP = tiempo estimado × prioridad × dificultad × urgencia × bienestar
ICU = carga total / capacidad disponible
```

### 3. Carga por colaborador

1. Abre el bloque de carga por colaborador.
2. Muestra un usuario en riesgo alto o crítico.
3. Explica que el sistema no sanciona; activa una alerta para revisión humana.

### 4. Tareas

1. Entra a Tareas.
2. Filtra por estado.
3. Crea una tarea nueva con “Sugerir automáticamente”.
4. Muestra que ASAP asigna según capacidad, rol/habilidad y carga actual.

### 5. Alertas y recomendaciones

1. Entra a Alertas y recomendaciones.
2. Abre una recomendación de reasignación.
3. Lee la explicación: persona sobrecargada, persona disponible, habilidad compatible y peso de la tarea.
4. Aplica la recomendación.

### 6. Auditoría

1. Entra a Auditoría.
2. Muestra el registro de la recomendación aplicada.
3. Explica trazabilidad y control humano.

### 7. Reportes

1. Entra a Reportes.
2. Exporta CSV.
3. Abre vista PDF e indica que se puede imprimir o guardar como PDF.

### 8. Entrada como empleado

1. Cierra sesión.
2. Entra como empleado.
3. Muestra que el empleado ve su propia carga y tareas, no información sensible de todos.

## Frases clave para el jurado

- “El aplicativo no automatiza decisiones laborales sensibles; entrega evidencia para revisión humana.”
- “La carga no se mide solo por número de tareas, sino por tiempo, dificultad, prioridad, urgencia y señales de bienestar.”
- “El MVP académico permite validar el flujo central: captura, cálculo, alerta, recomendación y reporte.”
- “La versión productiva debe validarse con datos reales anonimizados y línea base manual.”
